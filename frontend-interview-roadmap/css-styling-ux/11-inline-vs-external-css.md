
# 11. Inline vs. External CSS

## Quick Revision

There are three main ways to apply CSS styles to an HTML document:

*   **Inline CSS:** Styles applied directly to an HTML element using the `style` attribute.
*   **Internal CSS (Embedded CSS):** Styles defined within a `<style>` tag in the `<head>` section of an HTML document.
*   **External CSS:** Styles defined in a separate `.css` file and linked to the HTML document using a `<link>` tag.

## Detailed Explanation

### Inline CSS

*   **How it works:** Styles are applied directly to an HTML element using the `style` attribute.
*   **Example:** `<p style="color: blue; font-size: 16px;">This is a paragraph.</p>`
*   **Pros:**
    *   Highest specificity (overrides all other styles).
    *   Useful for quick, one-off style adjustments or dynamic styles generated by JavaScript.
*   **Cons:**
    *   **Not reusable:** Styles cannot be reused across multiple elements or pages.
    *   **Poor maintainability:** Mixes content and presentation, making code harder to read and manage.
    *   **No caching:** Styles are not cached by the browser.

### Internal CSS (Embedded CSS)

*   **How it works:** Styles are defined within a `<style>` tag in the `<head>` section of an HTML document.
*   **Example:**

    ```html
    <head>
      <style>
        h1 { color: red; }
        p { font-size: 14px; }
      </style>
    </head>
    ```

*   **Pros:**
    *   Useful for single-page applications or when styles are unique to a single page.
    *   Can override external stylesheets.
*   **Cons:**
    *   **Not reusable:** Styles are only available to the current HTML document.
    *   **No caching:** Styles are not cached by the browser.
    *   Increases HTML file size.

### External CSS (Recommended)

*   **How it works:** Styles are defined in a separate `.css` file and linked to the HTML document using a `<link>` tag in the `<head>` section.
*   **Example:**

    ```html
    <head>
      <link rel="stylesheet" href="styles.css">
    </head>
    ```

*   **Pros:**
    *   **Reusable:** Styles can be reused across multiple HTML documents.
    *   **Maintainable:** Separates content (HTML) from presentation (CSS), making code easier to read and manage.
    *   **Caching:** Browsers can cache external stylesheets, leading to faster page loads on subsequent visits.
    *   **Collaboration:** Easier for multiple developers to work on different parts of the styling.
*   **Cons:**
    *   Requires an additional HTTP request to fetch the stylesheet.

## Code Examples

### All Three Methods in One Page

```html
<!DOCTYPE html>
<html>
<head>
<title>CSS Styling Methods</title>
<link rel="stylesheet" href="external.css">
<style>
  /* Internal CSS */
  h1 {
    color: green;
  }
</style>
</head>
<body>

<h1 style="color: purple;">This is a Heading</h1> <!-- Inline CSS -->
<p>This is a paragraph.</p>

</body>
</html>
```

```css
/* external.css */
p {
  color: blue;
  font-size: 18px;
}
```

**Resulting styles (due to specificity and order):**

*   `<h1>`: purple (inline style has highest specificity)
*   `<p>`: blue, 18px (external CSS, assuming no other conflicting rules)

## When to Use Which

*   **External CSS:** Almost always the preferred method for most web development projects.
*   **Internal CSS:** Use sparingly, for page-specific styles or when you need to override external styles for a single page.
*   **Inline CSS:** Use very rarely, for dynamic styles generated by JavaScript or for quick, temporary overrides during development.
